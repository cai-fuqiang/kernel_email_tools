## 重新思考下项目初衷

### 目前学习内核痛点
1. 内核有非常广泛的知识面
   + 操作系统知识
   + 硬件API
   + 数据结构算法
2. 内核拥有非常久的历史
   + 30多年的历史
   + 非常多的commit, 以及邮件列表
3. 代码非常复杂，且部分模块多次重构，变动频繁
   + 仅靠代码看不懂代码，重要的信息隐藏在commit以及邮件列表的讨论中
4. 内核知识散落很大
   + commit message
   + mail list
   + lwn
   + Doc
   + 书籍
   + 大佬的博客


知识在于积累，而积累知识需要工具。而在AI浪潮下，AI成为了分析邮件代码
commit的重要助手，那么以何种方式驱使AI去做，是一个非常重要的
出发点.
